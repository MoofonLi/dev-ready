## Phase 1 — prune list + leak guard (FR-7, FR-9)

Smallest phase, highest user-visible value. Ships as v0.2.0.

- `manifest` module: add optional `prune` key (validation identical to `exclude`: non-empty relative-path strings, no leading `/`, no `..` segments). New field on `UpstreamPin`, default empty tuple.
- `fetch`: pass `exclude=pin.exclude + pin.prune` to `copier.run_copy` (order irrelevant to Copier; keep exclude first for readability).
- `manifest.json`: populate `prune` with the audited FR-7 list. NOTE for implementer: Copier exclude patterns are source paths relative to the template root; directory entries need the path form Copier matches (verify `img`, `hooks`, and individual `.github/workflows/*.yml` entries against a real run — the FR-7 list names intents, the implementer confirms exact patterns).
- `verify`: add `FORBIDDEN_PATHS = (".git", "copier.yml", "copier.yaml")`; `verify_project` raises `VerificationError` (exit 5) if any exists in the staged output. Do NOT add pruned paths here — prune is curation, not corruption; only leak-class paths are forbidden.
- Tests: manifest parsing (prune present/absent/invalid), fetch merge behavior (unit, no network), verify forbidden-path (tmp_path fixtures). CI generate-and-verify already exercises the real pipeline.
- Acceptance: a fresh generation contains none of the FR-7 pruned files; all FR-7 KEEP files still present; unit suite green without network.

## Phase 2 — project README (FR-8)

Depends on Phase 1 (prune must remove upstream README.md first).

- Add `README.md` to the `prune` list.
- New overlay template `templates/readme/README.md.tmpl`: project name heading, one-paragraph stack description, commands section mirroring CLAUDE.md.tmpl, link back to dev-ready and the upstream template. Always applied (like CLAUDE.md), not a selectable component.
- Overlay no-overwrite rule unchanged — prune guarantees no collision. If prune is somehow disabled and README.md exists, the existing `OverlayError` collision path is the correct behavior; do not special-case.
- Tests: overlay unit tests for the new file; template token check already generic.
- Acceptance: generated project has a README about ITSELF, not about the template; no broken `img/` references anywhere.
