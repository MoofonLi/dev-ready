# v0.4 Overview — dev-ready

Released: v0.4.0 | Status: Released
Scope: FR-15, FR-16, FR-17, FR-18

## What v0.4 shipped

dev-ready v0.4 introduces a complete vendoring infrastructure for external content alongside a major MIT content wave. External skills and documentation templates are now materialised directly in the repository with tracked 40-hex commit pins, zero-drift verification, and automated third-party notice management.

## Phase 1 — vendoring infrastructure (FR-15, FR-16)

- `manifest.json` `vendored` section: schema (`VendoredPin`: `repo`, `commit`, `license`, `paths`), loader validation (40-hex commit, path rules, duplicate rejection, dangling-reference check).
- `scripts/sync_vendored.py`: maintainer/CI tool that clones each vendored repo at its pinned commit and materialises snapshots under `src/dev_ready/templates/`.
- CI drift job (`vendored-drift` in `ci.yml`): byte-equality check via `sync_vendored.py --check`.
- Monthly bump workflow (`vendored-bump.yml`): resolves each vendored repo's HEAD, updates commits, opens bump PR.
- `scripts/bump_vendored.py`: stub script for the bump workflow.

## Phase 2 — MIT wave content + NOTICES automation (FR-17, FR-18)

- Four vendored sources materialised (`JuliusBrussee/caveman`, `mattpocock/skills` subset, `cloudflare/security-audit-skill`, `VoltAgent/awesome-design-md`). Skills catalog total: 7 (≤ 10 cap).
- Stamp schema bumped to `stamp_version: 2`: `.dev-ready.json` now records vendored pins for selected items.
- `scripts/check_notices_sync.py`: bidirectional CI check — every `vendored` manifest entry appears in `THIRD_PARTY_NOTICES.md`; no orphan NOTICES entries.
- `THIRD_PARTY_NOTICES.md` rewritten to include all four vendored sources with pinned commits.

## Notable decisions

- ADR-009: `manifest.json` `vendored` section with enforced provenance — now Implemented.
- ADR-008 integration modes: the four v0.4 sources all use `vendor` mode; `react-doctor` remains the sole `pinned-dependency` item.
- 10-skill catalog cap enforced: 7 skills total after v0.4.
- DESIGN.md templates from awesome-design-md ride inside the `docs` boolean component — no new catalog surface.
