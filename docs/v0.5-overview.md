# v0.5 Overview — dev-ready

Released: v0.5.0 | Status: Released
Scope: FR-19, FR-20

## What v0.5 shipped

dev-ready v0.5 is a content-only release riding entirely on v0.4's vendoring
machinery: an Apache-2.0 example-skill wave from anthropics/skills (with NOTICE
propagation into generated projects) and a set of Karpathy-derived development
guardrails folded into every generated `CLAUDE.md`.

## Phase 1 — anthropics Apache subset + NOTICE propagation (FR-19)

- Vendored two Apache-2.0 skills from `anthropics/skills` (pinned commit
  `1f630fdf9259cec4a14913127dfd7c3b69ef72eb`): `webapp-testing` and `frontend-design`.
  Skills catalog total: 9 (≤ 10 cap). Document-processing skills (docx/pdf/pptx/xlsx)
  remain permanently excluded.
- Apache NOTICE propagation: each vendored skill ships its own Apache-2.0 `LICENSE.txt`
  inside its snapshot, so the grant travels into every generated project that selects it.
- `scripts/check_notices_sync.py` extended to assert LICENSE-file presence in the
  snapshot of every Apache-2.0 `vendored` entry.

## Phase 2 — karpathy guardrails + close-out (FR-20)

- Folded a condensed, four-principle guardrail section (Think before coding, Simplicity
  first, Surgical changes, Goal-driven execution) into `templates/claude/CLAUDE.md.tmpl`,
  adapted from `multica-ai/andrej-karpathy-skills` (MIT, per its README).
- Integration mode: **adapted-rewrite** (attribution-only) — no `vendored` entry, no
  snapshot. `THIRD_PARTY_NOTICES.md` carries an attribution-only section citing
  "MIT, per README at 2c606141936f1eeef17fa3043a72095b4765b9c2".
- `scripts/check_notices_sync.py` learned to recognize an explicitly-marked
  attribution-only NOTICES entry (via the `Integration: adapted-rewrite` marker) so it
  is not treated as an orphan — a tested contract extension, not a weakening.
- Generated `CLAUDE.md` kept short (length budget enforced by test).

## Notable decisions

- ADR-008 integration modes: karpathy uses the adapted-rewrite (attribution-only) path,
  the first content integration that is neither vendor-snapshot nor pinned-dependency;
  consistent with ADR-008's existing "karpathy accepted as MIT per its README" note — no
  ADR status change.
- 10-skill catalog cap held: still 9 skills after v0.5 (FR-20 adds guardrail text, not a
  catalog item).
- No new runtime or dev dependencies in v0.5.
